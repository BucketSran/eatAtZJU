# Real Data Bootstrap

目标：把第一批餐厅数据从“虚构 demo seed”替换为“可追溯、可审核、可持续更新”的真实校园数据。

## 结论

不建议写自动爬虫直接抓取小红书/大众点评正文、图片、用户评论或绕过登录/风控的数据。更稳的冷启动方式是：人工调研 + 原创摘要 + 来源链接留档 + SQL 导入。

这样依然能利用小红书/大众点评作为参考来源，但不会把受平台条款和版权保护的内容直接复制进我们的数据库。

## 可采集内容

推荐采集事实性信息：

- 餐厅名、别名
- 位置区域、经纬度或大致校区方位
- 人均价格
- 菜系/场景标签
- 推荐菜品名和大致价格
- 是否适合午饭、夜宵、聚餐、外卖
- 参考链接和采集日期

需要原创整理：

- 推荐理由
- 学生短评
- 榜单描述
- 标签归纳

不要直接复制：

- 小红书/大众点评用户评论原文
- 平台图片
- 用户头像/昵称组合
- 需要登录后才能看到的非公开内容
- 任何通过绕过反爬、验证码、风控拿到的数据

## 工作流

1. 复制模板：

```bash
cp seed/real-restaurants.example.json seed/real-restaurants.json
```

2. 人工调研并填写 `seed/real-restaurants.json`。

3. 生成 SQL：

```bash
node scripts/generate-real-data-seed.cjs
```

4. 检查 `supabase/real-data-seed.sql`。

5. 在 Supabase SQL Editor 里执行。

6. 小程序重新编译，确认榜单和详情页显示真实餐厅。

## 数据质量规则

- 每个餐厅至少 1 个 `sourceRefs`。
- `reason` 必须是原创摘要，建议 60-120 字。
- 首批建议 30-50 家，不追求全量，优先覆盖紫金港北门、南门、西园、校内、三墩、城西银泰。
- 每家至少 2-4 个标签，标签尽量复用：`近`、`实惠`、`辣`、`不辣`、`夜宵`、`一人食`、`聚餐`、`校内`、`面食`、`甜品`、`快餐`、`外卖友好`。
- 如果来自学生投稿，把 `sourceRefs.platform` 写成 `student_ugc`。

## 后续自动化方向

- 用问卷星/腾讯问卷收集学生投稿，再人工审核导入。
- 接入小程序 UGC 提交表单，先进入审核队列。
- 对公开网页只做“链接收藏 + 人工摘要”，不自动复制正文。
- 如果未来获得商家或平台授权，再接入授权 API 或授权数据文件。
